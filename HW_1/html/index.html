Basic Git Commands

  • git config --global user.name - идентификация пользователя по имени, после этого каждое наше действие будет отмечено именем, таким образом, другие пользователи будут в курсе кто отвечает за какие изменения.
  • git config --global user.email blabal@bla - идентификация пользователя по e-mail, после этого каждое наше действие будет отмечено именем, таким образом, другие пользователи будут в курсе кто отвечает за какие изменения.
  • git init - инициализирует локальный git repository, после этого в указаной папке появляется скрытая папка .git в этой папке хранятся служебные файлы git.Простыми словами, включает Git для данного локальногот репозитория. (т.е папки)
  • git status - команда показывает состояние рабочего каталога и раздел проиндексированых файлов. При помощи ее можно проверить индексация и увидеть файлы которые не отслеживает Git.
  • git add - переносит указаный файл из рабочего католога в раздел индексированых файлов.
  • git commit - эта команда делает для проэкта снимок текущего состояния ИЗМЕНЕНИЙ, добавленных в раздел проиндексированых файлов. Иными словами записывает проиндексированые в репозиторий гит
  • git log - перечесляет все коммиты сделанные в репозитории в обратном хронологическом порядке - последние комыты нахолятся сверху.
  • git checkout - команда позволяющая переходить с ветки на ветку.
  • git branch - позволяет создавать, переиминовывать и удалять ветки.
  • git push - передает коммиты из локального репозитория в удаленныей.
  • git pull - загрузка проэкта из удаленного репозитория и обновления локальноготрепозитория этим содержимым.
  • git merge - позволяет склеивать отдельные ветки в одну единую ветку main.
  • git clone - как один ищ вариантов используется для клонирования существующего репозитория
  • gitignore file - создание и натсройка этого файла помогает указать какие файлы мы не хотим, что бы автоматически добавлялись в репозиторий или даже видеть их в списках неотслеживаемых. К таким файлам обычно относятся автоматически генерирумые файлы (временные файлы,различные логи, результаты сборки программ)

Advanced

git revert — это операция для безопасной отмены изменений, действие которой направлено в будущее. Для отката изменений команда не удаляет из истории коммиты или родительские элементы, a создает новый коммит с отменой нужных действий.

git rebase - Основная ее цель “перетаскивать” коммиты из одного места в другое. Также с помощью нее можно избавляться от ненужных коммитов, склеивать одни и менять порядок других.

git reset - универсальный инструмент для отмены изменений. Она имеет три основные формы вызова, соответствующие аргументам командной строки --soft, --mixed, --hard. Каждый из этих трех аргументов соответствует трем внутренним механизмам управления состоянием Git: дереву коммитов (HEAD), разделу проиндексированных файлов и рабочему каталогу.

git cherry-pick - ерёт изменения, вносимые одним коммитом, и пытается повторно применить их в виде нового коммита в текущей ветке. Эта возможность полезна в ситуации, когда нужно забрать парочку коммитов из другой ветки, а не сливать ветку целиком со всеми внесенными в нее изменениями

git stash - позволяет на время «сдать в архив» (или отложить) изменения, сделанные в рабочей копии, чтобы вы могли применить их позже. Откладывание изменений полезно, если вам необходимо переключить контекст и вы пока не готовы к созданию коммита.
